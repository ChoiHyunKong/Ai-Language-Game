# 📋 Q&A 및 확정 사항

## 1. 배포 플랫폼
**Q:** 배포는 Vercel 통해서 하는 것인가?

**A:** ✅ **Vercel** 사용 권장
- 무료 티어로 충분한 트래픽 지원
- GitHub 연동 시 자동 배포
- 향후 서버리스 함수 활용 가능

| 옵션 | 장점 | 단점 |
|------|------|------|
| GitHub Pages | 완전 무료, 설정 간단 | 정적 사이트만 가능 |
| **Vercel** ✅ | 무료 티어 충분, 빌드 자동화 | 트래픽 제한 있음 |

## 3. 게임 구조
**Q:** 게임의 종류는 산성비 / 퀴즈 방식 두 가지인가?

**A:** ✅ **산성비 엔진 + 3가지 모드**

```
🎮 게임 엔진: 산성비 스타일 (공통 시각 효과)
│
├── 📝 단어 연습 모드
│   └── 단어를 보고 그대로 입력
│
├── 🧠 의미 퀴즈 모드
│   └── 뜻을 보고 단어 유추하여 입력
│
└── ✏️ 문장 완성 모드
    └── 예문의 빈칸을 채워 입력
```

**확장성:** 새로운 모드 추가 가능한 모듈화 구조

## 4. 데이터 업데이트 방식
**Q:** CSV 형식으로만 업데이트가 가능한가?

**A:** ✅ **완전히 가능**
- 모든 퀴즈/단어 데이터는 `data/words.csv`에서 로드
- 코드 수정 없이 CSV만 교체하면 콘텐츠 업데이트
- PapaParse 라이브러리로 CSV 파싱

## 7. 랭킹 시스템
**Q:** 랭킹 시스템은 어떻게 적용되는가?

**A:** ✅ **2단계 구조**

| 구분 | 저장소 | 방식 |
|------|--------|------|
| 로컬 랭킹 | LocalStorage | 개인 최고 기록 저장 |
| 글로벌 랭킹 | Google Sheets API | 무료 DB로 활용 |

---

## 8. 데이터베이스
**Q:** DB 연동이 필요한가?

**A:** ❌ **전통적 DB 불필요**

| 데이터 유형 | 저장 방식 |
|-------------|-----------|
| 퀴즈/단어 데이터 | CSV 파일 |
| 개인 기록 | LocalStorage |
| 글로벌 랭킹 | Google Sheets (무료) |

👉 **서버리스 + 무료** 체계 유지

## 9. 개발 언어
**Q:** 어떤 언어를 사용하는가?

**A:** ✅ **순수 웹 기술 스택**

| 기술 | 용도 |
|------|------|
| **HTML5** | 구조 |
| **CSS3** | 스타일링, 애니메이션 |
| **Vanilla JavaScript (ES6+)** | 게임 로직 |

**의도적으로 프레임워크 미사용:**
- React, Vue 등 없이 순수 JS로 개발
- 가볍고 빠른 로딩
- 학습 목적에도 적합

# Q&A 및 확정 사항

최종 업데이트: 2025-12-23
프로젝트 기획 과정의 질의응답 및 결정 사항 기록.

---

## 1. 배포 플랫폼
- 배포처: Vercel 사용 권장
- 선정 사유: 무료 티어 지원, GitHub 연동 자동 배포, 향후 서버리스 함수 활용 가능

| 옵션 | 장점 | 단점 |
|------|------|------|
| GitHub Pages | 무료, 설정 간단 | 정적 사이트 위주 |
| Vercel | 빌드 자동화, 확장성 우수 | 트래픽 제한 존재 |

---

## 2. 형상 관리
- 플랫폼: GitHub 사용
- 운영 방식: Git 버전 관리, Vercel 자동 배포 연동, 이슈/PR 기반 협업

---

## 3. 게임 구조
- 방식: 산성비 엔진 기반 3가지 모드 구현
- 세부 모드: 
  - 단어 연습: 단어 그대로 입력
  - 의미 퀴즈: 뜻을 보고 단어 유추
  - 문장 완성: 예문 빈칸 채우기
- 확장성: 모듈화 구조를 통한 신규 모드 추가 용이성 확보

---

## 4. 데이터 업데이트 방식
- 방식: CSV 파일 기반 로드 (data/words.csv)
- 장점: 코드 수정 없이 콘텐츠 업데이트 가능, PapaParse 라이브러리 활용

---

## 5. 개발 폴더 구조
- public/: 정적 파일 (HTML, Manifest 등)
- src/: 소스 코드 (진입점, 엔진, 데이터 로더, AI 연동, UI 등)
- data/: 퀴즈 데이터 (words.csv)
- docs/: 개발 지침 및 가이드 문서

---

## 6. 확장성
- 설계: 모듈화된 클래스 구조 적용
- 추가 방법: BaseMode 상속 클래스 생성 후 GameEngine 등록

---

## 7. 랭킹 시스템
- 로컬 랭킹: LocalStorage (개인 최고 기록)
- 글로벌 랭킹: Firebase Realtime Database (전체 순위 공유)

---

## 8. 데이터베이스
- 구조: 서버리스 및 무료 환경 구축
- 분류: CSV(퀴즈 데이터), LocalStorage(개인 개인 기록), Firebase(글로벌 랭킹)

---

## 9. 개발 언어
- 기술 스택: HTML5, CSS3, Vanilla JavaScript (ES6+)
- 전략: 프레임워크 미사용 (로딩 속도 최적화, 경량화, 학습 목적)
